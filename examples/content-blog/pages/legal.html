This page's body is authored as raw HTML and rendered through @ssolu/mosaic-{astro,next}'s HTML pipeline (rehype-parse → rehype-sanitize → rehype-stringify).

Inline elements like bold, italic, and links survive sanitisation; scripts, inline event handlers, and <style> blocks do not.

What this proves

  • A record's body format is determined by the extension of the paired content file, not by content sniffing.
  • Markdown remains the default; HTML is an opt-in by saving the body as legal.html instead of legal.md.
  • The same record metadata (legal.json) drives the layout regardless of body format.

This is also why @type, title, and lede still render at the top of the page — the sidecar JSON handles the structured parts; the HTML file handles the prose.
