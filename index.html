The spec

Three rules. Nothing more.

RULE 01

A file is a record.

.json files hold structure. .md, .pdf, .png are opaque payloads. Metadata goes in a same-named .json sidecar.

# structured about.json → record # opaque + sidecar hello.md hello.json → sidecar
RULE 02

A folder is a collection.

Any directory is a collection. Folders nest freely. index.json inside a folder is that folder's record.

# team/ is a collection team/ index.json → team ada.json → team/ada grace.md grace.json → team/grace
RULE 03

The filename is the contract.

Names follow name[.modifier]*.ext. File form and folder form are the same identity — no rename, no broken refs.

# same identity: about about.json about/index.json → identical # locale modifier about.fr.md → locale fr

How it works

Reading a record is one pass.

STEP 1
Content
.json object, or empty for opaque files
↓
+
STEP 2
Sidecar merge
Sidecar JSON merged on top. Sidecar wins. Shallow.
↓
+
STEP 3
Cascade fill
Fills absent keys from ancestor defaults. locale by default; profiles declare more.
↓
→
STEP 4
Refs resolved
ref: strings replaced with targets. Single-pass, deterministic.
Fixed order. No recursion. Same folder → same result, always.

References

Link records with ref:

A reference is a JSON string starting with ref:. Two anchor forms, one optional JSON Pointer.

"ref:team/ada"
Absolute
Resolved from the root. Recommended default.
"ref:./other"
Relative
Resolved from the referring record's folder. Survives folder moves.
"ref:team/ada#/email"
JSON Pointer
RFC 6901 path after # selects a value inside the target.
