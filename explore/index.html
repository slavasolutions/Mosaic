One folder. Many sites.

The same content, rendered five ways.

Each card below is a real deployed Mosaic site. Same spec, same identity rules, same refs. The variation is the adapter (Astro or Next), the theme (default or dark), or the content folder (full or minimal). Pick one to see how a Mosaic folder feels under that combination.

Astro · default

Astro Default theme examples/content

The reference deploy. Astro Content Layer reads examples/content/, the design-tokens profile supplies the palette, and the web profile maps records to URLs under pages/.

Astro · dark

Astro Dark theme examples/content-dark

Same content, one line changed: defaults.theme: ref:/tokens/dark. Proves theme hot-swap — no template edit, no rebuild of the adapter, just a different ref.

Astro · minimal

Astro No tokens examples/content-minimal

A different folder with no tokens/ collection and three pages total. Demonstrates that the design profile is opt-in; the layout falls back to inline CSS defaults when the cascade has no theme record to resolve.
