# §7. Nommage et identité (Règle 3)

Variante française **sparse** : seule la prose est traduite. Le sidecar
correspondant (`naming.fr.json`) ne redéclare que `title` et `locale` ; tous
les autres champs (numéro de section, `see_also`, etc.) reviennent à la
version canonique `naming.json` via la résolution par variantes (Décision 3,
`ref:decisions/sidecar-variants`).

Chaque nom d'enregistrement DOIT correspondre à :

```
nom[.modificateur]*.ext
```

Les règles complètes restent celles de `ref:format/naming` (en).
