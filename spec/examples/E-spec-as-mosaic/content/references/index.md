# §11. References

This section defines **content references** only: a value in one record that
names another record. It does not define parent/child or membership
relationships — those are derived from the directory tree at zero cost and
MUST NOT be expressed as references. It does not define referential
integrity — the format defines resolution, not enforcement
(`ref:references/dangling`). It does not define query expressions of any
kind (`ref:references/hard-ceiling`).
